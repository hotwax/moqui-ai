# Archived plans

These are the historical, dated implementation plans (TDD build-logs) from the moqui-ai build-out, kept for provenance. They record *how* each feature was built, in order.

They are **not** a current reference. Many embed pre-refactor code and entity definitions — most notably the old `agentName`/`toolName` primary keys, which were superseded by the opaque `agentId`/`toolId` registry keystone. For the current as-built state, see [`../reference/`](../reference/) (entities, services, screens, configuration) and [`../explanation/`](../explanation/) (architecture, security model, decisions).
